Game UI
This section presents some concepts of the user UI.
The Grid
The grid is composed of:
Cells where initial all possible items are set.
For example in a 3x3 grid there is 3 items per cell in a total of 6 cells:
Lines, where each line has a category (example: faces, fruits, weapons ...)
For example in a 3x3 there is 3 lines:
And columns, that are used for expressing items relations.
For example in a 3x3 there is 3 columns:
Clues List
The clues list are located at the top:
There is 3 clues list: A, B, F.
You can switch to each clue list by tapping on the clues list button.
The only purpose of A and B clues list is to help you organize the clues. Its very simple when you are done with a clue you can send that clue to the other list by tapping on that clue, if clue is on list A then it will be sent to list B and vice-versa.
The F clue list, is used to filter clues by item, when this list is selected you can select any item on the grid, the list will show all clues containing the selected item,
Actions
The action buttons are located near the clues list,
they are mainly used to perform actions on the grid,
Expand the selected item on grid. When you are sure that a item belongs to grid cell, then you can expand it to the all cell.
if you think you made a mistake you can re-add a removed cell item. When you select a cell with removed items, the removed items will show with a gray color on left top side near clues and if you select one of the removed items this add button will appear and you can re-add the item.
Remove the selected item from the cell grid. If you are sure that item does not bellong on the grid cell you can remove it.
Switch to thinking mode, every changes in this mode will take effect. This mode helps you to highlight the grid items, so that you can visualize better the game grid on your own way.
